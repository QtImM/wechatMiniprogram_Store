根目录作为多语言、多进程聚合工程，按职责拆分为三个顶层子域：
- `shop-backend/`：Spring Boot 单体应用，采用 `shop-framework`（starter 自动装配）+ `shop-module-*`（业务模块）+ `shop-server`（启动入口）的 Maven 多模块分层；各 starter 通过 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 实现零配置注入。
- `shop-miniapp/`：原生微信小程序（wx-mall 迁移产物），通过 `utils/api.js` 中的 BASE_URL 指向后端 REST 接口，以 Token 认证与后端 `shop-starter-security` 对接。
- `sql/init.sql`：数据库初始化脚本由前后端共享，是双方数据契约的唯一来源。
前后端之间唯一的跨进程边界是 HTTP REST API，约定由后端 `CommonResult` 统一响应结构，小程序侧在 `api.js` 中封装请求拦截与错误码映射，形成“后端定义 → 前端消费”的单向契约。

采用「基础设施层 + 业务域模块 + 可执行入口」三层结构：
- 基础设施层 `shop-framework` 下拆分为 `shop-common`（通用 POJO/工具类）与多个 `shop-starter-*`（web、mybatis、security），通过 Spring Boot AutoConfiguration 对外暴露零配置能力；
- 业务域模块 `shop-module-member`、`shop-module-product`、`shop-module-system` 各自独立实现一个领域，仅依赖 framework 提供的 Starter，不互相引用；
- `shop-server` 作为唯一可执行 Jar，通过 `@ComponentScan("com.shop")` 与 `@MapperScan` 跨模块扫描 Controller/Service/Mapper，配合各 Starter 的 `AutoConfiguration.imports` 完成装配。
- 边界契约由 `shop-common` 统一：Controller 返回 `CommonResult`/`PageResult`，DAL 实体继承审计基类 `BaseDO`，全局异常处理在 starter 中集中拦截，从而在单体内部维持一致的 API 与持久化约定。

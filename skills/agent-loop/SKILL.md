# Agent Loop（项目版）

## 核心原则

以可观察的环境反馈驱动下一步，而不是按预设步骤盲目执行。每轮行动都必须让任务离验收标准更近，或提供足以改变策略的证据。

## 开始前

1. 明确用户目标、验收标准、授权范围和停止条件。
2. 读取仓库的 `AGENTS.md`、状态文档、规范与相关计划；这些约束优先于本 skill。
3. 写出当前已知事实与待验证假设。不要把假设当作事实继续执行。

## 循环

1. 选择当前风险最低、信息增益最大的最小行动。
2. 执行行动，并记录工具输出、测试结果、文件差异或页面状态等证据。
3. 将证据与验收标准比较：已完成、取得进展、无效、受阻或需要授权。
4. 已完成则做最终验证并汇报证据；取得进展则基于新事实进入下一轮；无效则换方法，不重复相同尝试；受阻或超出授权则向用户说明具体阻塞。

| 信号 | 下一步 |
| --- | --- |
| 结果满足验收标准 | 停止循环，给出验证证据，并更新项目状态。 |
| 结果缩小问题范围 | 用新事实选择下一项最小行动。 |
| 相同方法连续两次没有新增信息 | 改变信息源、工具或假设。 |
| 需要破坏性操作、外部发布或新增权限 | 先确认目标与授权。 |

## 完成与引用

不得因“已做了很多步骤”而结束；只在验收通过、用户要求暂停，或继续需要用户输入/授权时停止。

本 skill 以 OpenAI 的 [Unrolling the Codex agent loop](https://openai.com/index/unrolling-the-codex-agent-loop/) 为引用源：模型根据上下文选择最终答复或工具调用，工具结果回填上下文后继续推理，直至达到退出条件。

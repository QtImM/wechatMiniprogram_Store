# 微信小程序商城 — 全期实施方案

> 基于需求调研文档，结合当前项目进度（Plan1 骨架已完成），制定 P0 功能全覆盖的落地计划。

---

## 项目概况

| 维度 | 内容 |
|------|------|
| 行业 | 美妆护肤 + 食品零食 + 农产品特产 + 教育课程 + 医药保健 |
| 模式 | 自营 + 入驻混合（首期以自营为主，预留多商家扩展） |
| 目标 | 品牌展示/提升形象 + 会员体系管理 |
| 用户规模 | 首期 <1000人，DAU <100 |
| 客单价 | 200-500元 |
| 预算 | 50-100万 |
| 工期 | 2-3个月（目标 9月初上线） |
| 地域 | 本省区域 |

---

## 当前进度（已完成）

- ✅ Spring Boot 3.2 + Java 17 后端骨架（多模块 Maven 项目）
- ✅ 基础框架层（通用工具、Security JWT、MyBatis-Plus、Web 统一配置）
- ✅ 数据库核心表（会员、商品分类、SPU/SKU）
- ✅ Mock API 接口（首页7个接口 + 分类/商品/购物车/用户等）
- ✅ uni-app Vue2 小程序前端（35个页面，已跑通）
- ✅ Docker 部署配置

---

## 总体分期规划

```
Phase 1 (7月上旬)     核心交易闭环      ← 能下单、能支付、能发货
Phase 2 (7月中旬)     会员 + 营销体系    ← 付费会员、优惠券、满减
Phase 3 (7月下旬)     完整用户体验      ← 售后、评价、分享、消息
Phase 4 (8月上旬)     管理后台          ← PC端运营管理系统
Phase 5 (8月中旬)     联调测试 + 安全    ← 真机测试、安全加固
Phase 6 (8月下旬~9月) 部署上线 + 验收    ← 微信审核、灰度发布
```

## Phase 1：核心交易闭环（2周）

**目标**：用户能完成「浏览商品 → 加入购物车 → 下单 → 微信支付 → 商家发货 → 确认收货」全流程。

| 佣金提现 | 满额可提现到微信零钱（企业付款到零钱） |

### 2.6 买赠 + 组合优惠（2天）

| 任务 | 说明 |
|------|------|
| 买A赠B | 购买指定商品自动添加赠品（0元商品） |
| 组合套装 | 多商品打包销售，套装价低于单买总价 |
| 前端展示 | 商品详情标注"买即赠"/"组合优惠" |

### 2.7 价格计算引擎（贯穿）

订单价格计算链（按优先级）：
```
原价 → 会员折扣 → 满减优惠 → 优惠券抵扣 → 运费 = 实付金额
```

规则：
- 会员折扣和满减**不叠加**，取优惠大的
- 优惠券可与以上叠加
- 分销佣金基于实付金额计算
